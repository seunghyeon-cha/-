# 예림투어 프로젝트 팀 구조 및 운영 규칙

## 팀 구조

### 1. 프론트엔드팀 (`/frontend`)
- **담당**: UI/UX 구현, React/Next.js 개발, 상태 관리
- **작업 범위**: `/frontend` 폴더 내 모든 파일
- **작업 기록**: `/frontend/WORK_LOG.md`

### 2. 백엔드팀 (`/backend`)
- **담당**: API 개발, 데이터베이스 관리, 인증/인가
- **작업 범위**: `/backend` 폴더 내 모든 파일
- **작업 기록**: `/backend/WORK_LOG.md`

### 3. 디자인팀 (`/design`)
- **담당**: UI 디자인, 디자인 시스템, 에셋 관리
- **작업 범위**: `/design` 폴더 내 모든 파일
- **작업 기록**: `/design/WORK_LOG.md`

---

## 운영 규칙

### 핵심 원칙
1. **독립성**: 각 팀은 자신의 폴더 내에서만 작업
2. **비침범**: 다른 팀 폴더의 파일을 직접 수정하지 않음
3. **기록 필수**: 모든 작업은 완료 후 WORK_LOG.md에 기록
4. **추적 가능**: 모든 변경사항은 추후 수정/검토 가능해야 함

### 작업 기록 양식
```markdown
## [날짜] 작업 내용

### 작업자
- 팀: [프론트엔드/백엔드/디자인]

### 작업 내용
- 완료된 작업 목록

### 변경된 파일
- 파일 경로 목록

### 테스트 결과
- 테스트 통과 여부

### 비고
- 추가 참고사항
```

### 팀 간 협업
- API 연동: 프론트엔드 ↔ 백엔드 (API 명세서 기반)
- 디자인 적용: 프론트엔드 ← 디자인 (디자인 토큰/에셋 기반)
- 협업 필요시 PO에게 요청
