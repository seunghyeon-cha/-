4. **지도 뷰 (선택적)**
   - 토글로 지도/리스트 전환

**기능 요구사항**:
- URL 쿼리 파라미터로 필터 상태 관리
- 필터 변경 시 API 호출
- 북마크 토글 (로그인 필요)
- SEO 최적화

   - 장소명

   - 평점 (별 + 숫자) + 리뷰 수
   - 주소 (클릭 시 지도로 이동)

   - 웹사이트 링크
   - 영업시간
   - 북마크 버튼

4. **지도**
   - Kakao Map 임베드

5. **리뷰 섹션**
   - 평점 분포 차트 (5점~1점 비율)
   - 정렬 (최신순, 평점순, 좋아요순)
   - 리뷰 카드:
     - 사용자 프로필 이미지 + 이름
     - 별점
     - 작성일
     - 리뷰 내용
     - 리뷰 이미지 (있을 경우)

   - 페이지네이션

6. **리뷰 작성 모달**
   - 별점 선택
   - 텍스트 입력
   - 이미지 업로드 (최대 5장)
   - 제출 버튼

7. **관련 장소 추천 (하단)**
   - 같은 카테고리의 다른 장소 3~4개

**기능 요구사항**:
- SSR 또는 SSG (메타 태그 SEO)
- 리뷰 무한 스크롤
- 로그인 여부에 따른 버튼 표시
- 북마크/좋아요 토글

**산출물**:
- 장소 목록 페이지

### 디자인 원칙
1. **일관성**: 모든 페이지에서 동일한 컴포넌트 스타일 사용

### 컬러 팔레트 (디자인팀과 협의)
- Primary: 여행/모험을 연상시키는 색 (예: 파란색, 오렌지)
- Secondary: 보조 색상
- Gray Scale: 텍스트, 배경
- Success: 초록색
- Error: 빨간색
- Warning: 노란색

### 타이포그래피
- 제목: 24px ~ 48px (Bold)
- 본문: 14px ~ 16px (Regular)
- 캡션: 12px (Light)
- 한글 폰트: Pretendard 또는 Noto Sans KR

### 간격 (Spacing)
- xs: 4px
- sm: 8px
- md: 16px
- lg: 24px
- xl: 32px
- 2xl: 48px

### 브레이크포인트
- Mobile: 0 ~ 767px
- Tablet: 768px ~ 1023px
- Desktop: 1024px ~ 1439px
- Large: 1440px+

## 🔧 개발 규칙

### 컴포넌트 작성
- 함수형 컴포넌트 사용
- Props는 TypeScript 인터페이스로 정의
- 재사용 가능하게 설계

### 네이밍 규칙
- 컴포넌트: PascalCase
- 함수/변수: camelCase
- 상수: UPPER_SNAKE_CASE
- 파일명: kebab-case 또는 PascalCase (컴포넌트)

### CSS 규칙
- Tailwind CSS 유틸리티 클래스 우선
- 복잡한 스타일은 CSS Module
- 전역 스타일은 최소화

### Git 커밋 메시지
```

style: UI 스타일 변경
```
