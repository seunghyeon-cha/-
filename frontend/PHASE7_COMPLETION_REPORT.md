## 🔧 기술적 결정 사항

### 1. 레이아웃 구조: 사이드바 네비게이션

**선택 이유**:
- 명확한 계층 구조
- 많은 메뉴 항목 수용 가능
- 데스크톱 환경에 적합

**구현**:
- Next.js App Router의 `layout.tsx` 활용
- Sticky 사이드바 (스크롤해도 고정)
- 섹션별 그룹화 (내 정보, 내 활동, 관리, 설정)

### 2. 알림 설정: 로컬 스토리지 사용

**배경**: 백엔드에 알림 API 없음

**해결책**: 로컬 스토리지로 임시 구현
```typescript
const STORAGE_KEY = 'notification_settings';
localStorage.setItem(STORAGE_KEY, JSON.stringify(settings));
```

**장점**:
- 즉시 구현 가능
- 사용자 경험 제공
- 향후 API 연동 쉬움 (인터페이스 준비됨)

**한계**:
- 여러 기기 간 동기화 불가
- 실제 알림 기능 작동 안 함

### 3. 검색 및 필터링: 프론트엔드 처리

**배경**: API에 검색 기능 없음

**해결책**: 클라이언트 사이드 필터링
```typescript
const filteredBoards = boards.filter((board) => {
  const query = searchQuery.toLowerCase();
  return (
    board.title.toLowerCase().includes(query) ||
    board.content.toLowerCase().includes(query)
  );
});
```

**장점**:
- 즉시 반응
- 백엔드 부하 없음

**단점**:
- 현재 페이지 데이터만 검색 가능
- 대량 데이터 시 성능 저하 가능

**향후 개선**: 백엔드 검색 API 추가

### 4. 페이지네이션: 10-12개씩

**결정**:
- 리스트 형태: 10개씩 (boards, reviews)
- 그리드 형태: 12개씩 (bookmarks, 3x4 그리드)

**이유**:
- 적절한 정보량
- 로딩 시간 최적화
- 그리드는 3의 배수로 깔끔한 레이아웃

## 🎨 UI/UX 개선 사항

### 1. 로딩 상태

**스켈레톤 UI 사용**:
```tsx
<div className="h-6 bg-gray-200 rounded w-3/4 mb-3 animate-pulse"></div>
```

- 컨텐츠 모양 유지
- 부드러운 로딩 경험
- 스피너보다 프로페셔널

### 2. 빈 상태 (Empty State)

**구성**:
- 큰 아이콘 (이모지 사용)
- 명확한 메시지
- 액션 버튼 (다음 행동 유도)

**예시**:
```tsx
<div className="text-center py-20">
  <div className="text-6xl mb-4">📝</div>
  <h3>작성한 게시글이 없습니다</h3>
  <p>첫 게시글을 작성해보세요</p>
  <Link href="/boards/new">
    <button>첫 게시글 작성하기</button>
  </Link>
</div>
```

### 3. 호버 효과

**북마크 이미지**:
```tsx
className="group-hover:scale-105 transition-transform duration-300"
```

**카드 그림자**:
```tsx
className="hover:shadow-md transition-shadow"
```

### 4. 통계 정보

**아이콘 + 숫자**:
```tsx
<span>👁 120</span>  // 조회수
<span>👍 15</span>   // 좋아요
<span>💬 8</span>    // 댓글
```

- 직관적
- 공간 절약
- 국제적

### 5. 반응형 디자인

**그리드**:
```tsx
grid-cols-1 md:grid-cols-2 lg:grid-cols-3
```

**레이아웃**:
```tsx
lg:col-span-1  // 사이드바 (1/4)
lg:col-span-3  // 컨텐츠 (3/4)
```
